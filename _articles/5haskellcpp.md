# What is the most fun programming language?

I pose this question because I seldom hear the answer. We always talk about the best programming language to get hired or what is the best programming language to do X (see any programming conversation.)

Let me be clear here: I am not saying that these two languages are the best languages. I am just saying that these two languages are the most fun to play around with (for me.) This is a very subjective list and to be honest, the real takeaway from this article is for you think about what is the most fun programming language for you.

First, we have to define what "programmers" are. For this 🌶️ take, I see a programmer as an artist / craftsmen. Maybe it's because of my upbringing as an artist but I don't really look at programming as an engineering thing: that's what software engineering is all about. Engineering is pragmatic, where you got to think about trade offs and stuff. But if programming is akin to an artistic skill like painting, you want to ideally stretch yourself cause doing the same thing again and again is boring. 

With that in mind, we want to code in languages that **stretch** you. That means to expose you to as many new things, cause exploring is fun.

I want to be very clear: this is my perspective and what I do for myself. If you find something else works, that's great. Don't just go and do this because I said so. Make up your own mind and decide what's right for your fun.

# Enter: Haskell and C++

These two languages are literally the polar opposites of each other - one language is *elegant and flowery* - you sure as shit can't do basic IO without a "monad" because "side-effects bad" but it looks like the most beautiful thing once you are done. The other language is Dark Souls The Programming Language. You will get punished for not managing your memory and things will come down to a screeching halt if you don't care enough. 

From my experience, literally every language that I am drawn to as a programmer sits somewhere between the two. Java has a nice C-like syntax, JavaScript has some functional elements, TypeScript is evidently very influenced by the Haskell Type system. You can clearly see the inspirations taken from these two but because they are so different from each other, they provide that incredible experience to try out new paradigms, ideas etc. 

These two languages are simply the most fun because of that ability to play around with them. And to be honest, that's the joy of it all.
